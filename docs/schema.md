# Data Contracts

This is the cross-agent contract. Every service codes against these two schemas:
the **topology.json** shape (Controller's domain) and the **InfluxDB measurements**
(time-series domain). Change them here first, then update consumers.

---

## 1. `topology.json` — Source of Truth

Path: `dev-env/config/topology.json` (generated by `dev-env/config/generate_topology.py`).
Mounted read-write to the Controller, read-only to all DU/CU simulators. Only the
Controller writes it, atomically (`.tmp` → rename).

```jsonc
{
  "metadata": {
    "deployment": "Malleswaram, North Bangalore",
    "operator_market_share": 0.40,
    "active_ues_peak": 18400,
    "cell_level_max_ues": 16500,
    "site_count": 10,
    "cell_count": 30
  },
  "cus": {
    "CU-MLS": { "cu_id": "CU-MLS", "area": "Malleswaram", "lat": 13.0050, "lon": 77.5700 }
  },
  "dus": {
    "DU-MLS-1": { "du_id": "DU-MLS-1", "cu_id": "CU-MLS", "area": "north",
                  "lat": 13.0123, "lon": 77.5639, "max_cells": 12 }
  },
  "cells": {
    "MLS_RWS_01": {
      "cell_id": "MLS_RWS_01",      // MLS_<SITE>_<SECTOR>
      "du_id": "DU-MLS-1",
      "cu_id": "CU-MLS",
      "area": "Malleswaram Railway Station",
      "lat": 13.0129, "lon": 77.5571,
      "generation": "5G",            // 5G | 4G
      "band": "n78",                 // n78 | n41 | B40 | B3
      "freq_mhz": 3500,
      "vendor": "Nokia",             // Nokia | Ericsson | Samsung | ZTE
      "hardware_model": "AirScale MAA 64T64R",
      "antenna_config": "64T64R",    // 64T64R (5G) | 4T4R (4G)
      "pci": 1,                      // unique; planner uses graph-coloring
      "tx_power_w": 200,
      "idle_power_w": 120,
      "peak_dl_mbps": 3800,
      "max_ues": 900
    }
  }
}
```

**Per-band radio parameters** (cell-level `max_ues` sums to 16,500):

| band | gen | freq MHz | max_ues | tx_power_w | idle_power_w | antenna |
|------|-----|---------:|--------:|-----------:|-------------:|---------|
| n78  | 5G  | 3500 | 900 | 200 | 120 | 64T64R |
| n41  | 5G  | 2500 | 700 | 160 | 120 | 64T64R |
| B40  | 4G  | 2300 | 300 |  80 |  50 | 4T4R   |
| B3   | 4G  | 1800 | 250 |  80 |  50 | 4T4R   |

**Vendor distribution** (per-site, 9/9/6/6 ≈ 25% each — see generator note on the
spec's "10 cells per vendor" not dividing evenly into 30):

| vendor | sites | cells | 5G peak DL | 4G peak DL |
|--------|-------|------:|-----------:|-----------:|
| Nokia    | RWS, SPG, MGR | 9 | 3800 | 220 |
| Ericsson | 18C, 3MN, CHD | 9 | 3600 | 200 |
| Samsung  | BEL, 10C      | 6 | 3400 | 190 |
| ZTE      | SNK, 6CR      | 6 | 3200 | 180 |

**DU / CU grouping**: CU-MLS → DU-MLS-1 (north: RWS, 18C, BEL, SNK = 12 cells),
DU-MLS-2 (central: SPG, 3MN, 10C = 9), DU-MLS-3 (south-west: MGR, CHD, 6CR = 9).

## 2. InfluxDB Measurements

Org `telecom`, bucket `telecom_metrics`. Tags are indexed; fields are values.

| Measurement | Tags | Key fields |
|---|---|---|
| `cell_kpi` | cell_id, area, band, pci, du_id, cu_id, vendor, generation | connected_ues, dl_throughput_mbps, ul_throughput_mbps, rsrp_dbm, rsrq_db, sinr_db, power_w, prb_dl_pct, prb_ul_pct, packet_loss_pct, cqi, mcs, bler_pct, latency_ms, jitter_ms, interference_dbm |
| `du_kpi` | du_id, cu_id | active_ues, cell_count, cpu_pct, memory_pct, fronthaul_latency_us, processing_delay_ms, f1_msg_per_sec |
| `cu_kpi` | cu_id | du_count, rrc_connected, rrc_idle, rrc_setup_rate, inter_du_ho_rate, pdcp_dl_gbps, pdcp_ul_gbps, f1_latency_ms, n2_latency_ms, n3_latency_ms, e1_latency_ms, cpu_pct, memory_pct |
| `core_kpi` | component, instance_id | **AMF**: registered_ues, active_sessions, nas_msg_per_sec, paging_per_sec, handover_per_sec · **SMF**: active_pdu_sessions, session_setup_rate, ip_pool_utilization_pct · **UPF**: dl_throughput_gbps, ul_throughput_gbps, active_tunnels, packet_drop_rate |
| `ue_mobility` | ue_id, source_cell, target_cell, event_type | rsrp_source, rsrp_target, ho_duration_ms, velocity_kmh |
| `ue_usage` | ue_id, cell_id, slice_type | dl_bytes, ul_bytes, latency_ms, jitter_ms, packet_loss |
| `alerts` | severity, cell_id, du_id, alert_type | message, metric_value, threshold, ai_confidence |
| `son_actions` | cell_id, du_id, action_type | message, confidence |
| `topology_event` | event_type | cell_id, du_id, from_component, to_component |

**Enumerations**
- `severity`: INFO | WARNING | CRITICAL
- `alert_type`: OVERLOAD | UNDERLOAD | SINR_DEGRADATION | POWER_WASTE | LOAD_BALANCE
- `action_type`: LOAD_BALANCE | TRAFFIC_STEER | PCI_REOPT_REQUEST | DTX_RECOMMEND
- `slice_type`: eMBB | URLLC | mMTC
- `event_type` (mobility): HANDOVER | ATTACH | DETACH
- `event_type` (topology): CELL_MOVE | DU_MOVE | CELL_ADD | CELL_REMOVE | TOPOLOGY_REPLACE
- `component` (core): AMF | SMF | UPF

**BiLSTM feature vector** (9 features, `cell_kpi` fields, order fixed in `ml/model.py`):
`prb_dl_pct, sinr_db, connected_ues, power_w, packet_loss_pct, dl_throughput_mbps, cqi, bler_pct, latency_ms`

**Classes** (5): NORMAL(0) OVERLOAD(1) UNDERLOAD(2) SINR_LOW(3) POWER_WASTE(4)
